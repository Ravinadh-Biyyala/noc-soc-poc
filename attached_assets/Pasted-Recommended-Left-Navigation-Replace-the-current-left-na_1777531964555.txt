### 2. Workspaces

Purpose: Organize data, dashboards, joins, glossary, and reports by business use case.

Example workspaces:

* Broker Performance Analysis
* Sales Performance
* HR Workforce Analytics
* Finance Review
* Claims Risk Analysis

Each workspace contains:

* uploaded files
* prepared datasets
* joins
* dashboards
* saved insights
* reports
* glossary
* governance status

This is very important. Without workspaces, the app feels like a one-time dashboard generator.

### 3. Data Uploads

Purpose: Upload Excel/CSV files.

Features:

* Drag-and-drop file upload
* Multi-file upload
* File preview
* Sheet selection for Excel
* Auto-detect headers
* Auto-detect data types
* Detect date, currency, percentage, ID, category, measure fields
* Show upload status
* Show file metadata

After upload, route user to **Data Understanding**.

---

### 4. Data Preparation

Purpose: Let Gen-BI understand, clean, join, and prepare data.

This should have sub-sections:

#### A. Data Understanding

Show:

* detected columns
* sample rows
* inferred business meaning
* dimension/measure classification
* possible KPI fields
* possible date fields
* possible ID fields

#### B. Data Quality

Show:

* missing values
* duplicates
* inconsistent formats
* outliers
* invalid dates
* empty columns
* suspicious records

Add a **Data Readiness Score**.

Example:

> Data Readiness Score: 82/100
> 12 missing commission values
> 4 duplicate policy records
> 7 inconsistent date formats

Actions:

* Auto-fix
* Ignore
* Review manually
* Download issue report

#### C. Join Studio

This is a major wow feature.

Show uploaded datasets visually as cards.

For each file, show:

* file name
* row count
* column count
* detected keys
* data quality score

Gen-BI should suggest joins.

Example:

> Suggested Join: Policy Data + Commission Data
> Join Key: Policy Number
> Confidence: 96%
> Match Rate: 94%
> Unmatched Records: 312

Support:

* exact joins
* fuzzy joins
* multi-column joins
* lookup joins
* date-range joins

Actions:

* Accept join
* Modify join
* Preview result
* Save prepared dataset

#### D. Metric Builder

Purpose: Define or approve KPI formulas.

Example:

* Written Premium = SUM(premium_amount)
* Commission Revenue = SUM(commission_amount)
* Renewal Rate = renewed policies / eligible policies
* Loss Ratio = claims paid / premium earned

Add status:

* AI Suggested
* User Approved
* Certified

### 6. Ask Gen-BI

Purpose: Natural language analytics.

This should not only be a side panel. It should also be a full-screen experience.

User can ask:

* “What was total commission revenue in 2024?”
* “Why did renewal rate drop in Q3?”
* “Show broker-wise premium growth.”
* “Create a dashboard for product performance.”
* “Generate a report for executive review.”

For every answer, show:

* direct answer
* formula used
* filters applied
* source dataset
* confidence score
* chart preview
* button to create dashboard
* button to save insight

Example response:

> Total commission revenue for 2024 is **$31.8M**.
> Formula used: SUM(commission_revenue)
> Filter applied: Year = 2024
> Source: Broker_Commission.csv
> Confidence: High

Actions:

* View chart
* Create dashboard
* Save insight
* Export answer
* Ask follow-up

# 5. The Most Important Wow Features

Do not build too many random features. Prioritize these.

## Wow Feature 1: Smart Data Understanding

After upload, Gen-BI should explain the data like an analyst.

Example:

> This dataset appears to contain insurance policy and commission information. I detected premium, broker, policy, product, region, renewal, and claims-related fields.

---

## Wow Feature 2: Data Readiness Score

Before dashboard generation, show a score.

Example:

> Data Readiness Score: 84/100

This immediately creates trust.

---

## Wow Feature 3: Intelligent Join Studio

This is one of the most important features.

The system should recommend joins and explain confidence.

Example:

> Policy Data and Commission Data can be joined using Policy Number with 96% confidence.

---

## Wow Feature 4: Auto KPI Detection

System should detect KPIs automatically.

Example:

> I detected 8 possible KPIs: Written Premium, Commission Revenue, Policies Bound, Renewal Rate, Loss Ratio, Quote-to-Bind, Average Premium, Broker Contribution.

---

## Wow Feature 5: Question-to-Dashboard

User asks a question. The system creates a dashboard.

Example:

> Show me why commission revenue increased in 2024.

System creates:

* trend chart
* broker contribution
* product contribution
* region split
* explanation
* recommended actions

---

## Wow Feature 6: Explain Every Answer

Every AI answer must include:

* formula
* filters
* source file
* confidence
* generated chart

This is mandatory for enterprise adoption.

---

## Wow Feature 7: Insight Cards

System should automatically create insight cards.

Example:

> Top 5 brokers contribute 61% of written premium.

Each insight card should be saveable.

---

## Wow Feature 8: Trust Panel

Every dashboard should have a trust panel.

Show:

* source files
* joins used
* filters applied
* metric formulas
* data quality score
* last refresh date

---

## Wow Feature 9: Report Generation

User should convert dashboards into PDF/PPT.

Example:

> Generate a 5-slide executive pack from this dashboard.

---

## Wow Feature 10: Workspace Memory

The workspace should remember:

* uploaded files
* approved joins
* approved metrics
* saved dashboards
* saved insights
* reports

This makes the app useful beyond a single session.
